🔐 Authentication & SSO Layer

Duende IdentityServer handles OIDC. Each component is single-instance (LB restricted to one node due to operational data not being shared). SAML 2.0 SP handles federated SSO for 40+ partners. Login app redirects to SSO/OIDC flow.

Limitation: Duende cannot share operational data across servers. If a token request starts on server1, the LB sends to server2 which fails. Load balancer is restricted to a single instance per component.

🔄 Data Processing Flow (ETL Pipeline)
1
Master Import
EMSDBHA (EMS) → PEMSDBSQL01\UEG
Runs ES_Initial_MasterImport and ES_Daily_Incremental_Delete_Load to populate DataLoader and Client databases.
2
Daily Vendor Import
EMSDBPR06 → PEMSDBSQL01 (EMSUPortalCommon)
EMSDBPR11 Daily Vendor Import process feeds vendor data into the portal common database.
3
EcoTrak Manual Process
Manual Input → EMSDBPR05 (UtilitymartSub)
EcoTrak data is manually entered into the UtilitymartSub database.
4
Utility Data Sync
EMSDBPR05 (UtilitymartSub) → PEMSDBSQL01 (DataLoader)
Utility data syncs from the UtilitymartSub database into the central DataLoader.

Production Deployment Steps (Manual)

  • Request privilege escalation via ADO ticket on EMS-IBIS board + privpromotion.cassibisint.com
  • Log into target Mako server, open IIS Manager, navigate to ExpenseSmart site
  • Create new release folder, paste published code, copy web.config from previous release
  • Ensure workflow folder retains .aspx.vb code-behind files (not compiled pages)
  • Update "Physical Path" in IIS basic settings to new release folder
  • Restart site, verify in browser, then move back into load balancer
  • Note: WebDev01New build cannot be used directly for prod (DevExpress license issue)
